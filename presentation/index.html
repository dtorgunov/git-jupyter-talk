What is git and why should you care?

  • Flexible version control
  • Curation of the "definitive" version
  • Clear timeline of changes (include "blame" screenshot)

Who changed what?

Some vocabulary:

  • Commit: a logically related set of changes
  • Repository (repo): a collection of commits
  • Branch: a dynamic pointer to a specific commit
  • Tag: a static pointer to a specific commit
